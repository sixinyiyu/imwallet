# IMWallet API 版本管理策略

## 当前版本

所有 API 挂载在 `/api/v1` 下。

## v1 兼容承诺

### 可以做的（向后兼容）

- **新增响应字段**：v1 响应体可以新增字段，前端应忽略未知字段（JSON 解析天然兼容）
- **新增 API 路径**：在 `/api/v1` 下新增接口路径
- **新增查询参数**：现有接口可以新增可选查询参数，不影响已有调用

### 不能做的（破坏兼容）

- **删除响应字段**：已有字段不能删除或改名
- **改变字段类型**：如 `String` → `Number`、`Object` → `Array`
- **改变请求必填参数**：已有可选参数不能变为必填
- **删除 API 路径**：已有接口路径不能删除
- **改变 HTTP 方法**：如 GET → POST
- **改变错误响应结构**：`ErrorResponse` 的 `code` 和 `message` 字段不能删除或改名

## 版本升级流程

当需要做破坏兼容的改动时：

1. 新建 `/api/v2` 路由，与 v1 共存
2. v1 至少保留 **6 个月**并行期
3. 在 v1 响应中添加 `deprecated: true` 标记（可选）
4. 前端逐步迁移到 v2
5. 并行期结束后，v1 路由可标记为 `deprecated` 但仍保留
6. 最终移除 v1 需提前 **30 天**通知所有客户端

## 错误响应格式（v1）

```json
{
  "code": "WALLET_NOT_FOUND",
  "message": "钱包不存在"
}
```

- `code`：业务错误码（机器可读），用于前端精细化处理
- `message`：人类可读的错误描述

## 前端适配建议

- 解析响应时只取需要的字段，忽略未知字段
- 错误处理优先根据 `code` 字段，`message` 仅用于展示
- 新增字段上线后，前端无需同步更新即可正常工作
